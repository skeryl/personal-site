# Share Entry

Create a pull request that shares a journal entry between forked repos without causing merge conflicts.

## Background

This site is forked across collaborators. Each fork has its own curated set of entries. The `post-summarizer` Vite plugin automatically generates `entries/index.ts` and all route files under `routes/journal/` at dev/build time. Because of this, a PR to share an entry should **only include source files** — never generated files.

## What to include in the PR

A journal entry consists of:

1. **Entry definition** (required): `packages/ui/src/lib/entries/{post-id}.ts`
   - Imports and re-exports a `Post` object with summary metadata and a content reference
   - Uses `PostType.exploration`, `PostType.experiment`, or `PostType.experiment3d` from `@sc/model`

2. **Content component(s)** (required): one of:
   - `packages/ui/src/lib/explorations/{post-id}.svelte` (simple exploration)
   - `packages/ui/src/lib/explorations/{post-id}/` (multi-file exploration with subdirectory)
   - Inline content class in the entry `.ts` file (for `experiment` / `experiment3d` types using `grraf` or Three.js)

3. **Assets** (if any): videos, images, or data files referenced by the content component

## What to NEVER include

- `packages/ui/src/lib/entries/index.ts` — auto-generated by `post-summarizer`
- `packages/ui/src/routes/journal/{post-id}/` — auto-generated by `post-summarizer`
- `packages/ui/src/lib/assets/videos/posts/index.ts` — auto-generated by `post-summarizer`
- Changes to shared components (Nav, PostList, Post, Tags, etc.) unless explicitly intended
- Changes to `theme.css` or `tailwind.config.ts`

## Entry file template

```typescript
import { type Post, PostType } from '@sc/model';
import MyEntry from '$lib/explorations/{post-id}.svelte';

const post: Post = {
  summary: {
    id: '{post-id}',
    tags: [/* relevant tags */],
    title: '{Title}',
    subtitle: '{optional subtitle}',
    timestamp: new Date(/* year, month (0-indexed), day */),
    type: PostType.exploration,
    isHidden: false,
    collaborators: [/* { name, role, url? } */]
  },
  content: () => MyEntry
};

export default post;
```

## Steps

1. Identify the entry file(s) and content component(s) to share
2. Ensure the entry `id` doesn't collide with an existing entry in the target repo
3. Create a branch and add **only** the source files listed above
4. Create the PR targeting the receiving repo's `main` branch
5. The PR description should note which entry is being shared and list the files included

## Verification

After the PR is merged, the receiving repo just needs to run `npm run dev` or `npm run build` — the `post-summarizer` plugin will detect the new entry file and auto-generate the index and route files.
